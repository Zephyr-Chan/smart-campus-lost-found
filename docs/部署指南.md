# 校园失物招领系统 - 部署指南

## 一、本地开发环境部署

### 1.1 前置要求

- JDK 17+
- Maven 3.8+
- MySQL 8.0+
- Redis 6.0+（推荐 Redis Stack 支持向量检索）
- Node.js（可选，用于前端工具）

### 1.2 数据库初始化

```bash
# 登录 MySQL
mysql -u root -p

# 创建数据库
CREATE DATABASE lost_found DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 执行初始化脚本
USE lost_found;
SOURCE src/main/resources/sql/init.sql;
SOURCE src/main/resources/sql/migration_v2.sql;
```

### 1.3 配置修改

修改 `src/main/resources/application.yml`：
- 数据库连接地址/用户名/密码
- Redis 地址
- 文件上传路径
- AI 服务地址（可选）
- 高德地图 Key（可选）
- 邮件配置（可选）

### 1.4 启动应用

```bash
# 编译
mvn clean compile

# 运行
mvn spring-boot:run

# 访问
http://localhost:8080
```

默认管理员账号：`admin` / `debug123`

## 二、Python AI 微服务部署（可选）

### 2.1 环境准备

```bash
cd ai-service

# 创建虚拟环境
python -m venv venv
source venv/bin/activate  # Linux/Mac
# venv\Scripts\activate   # Windows

# 安装依赖
pip install -r requirements.txt
```

### 2.2 启动服务

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8001 --reload
```

### 2.3 验证

```bash
curl http://localhost:8001/api/ai/health
# {"status": "up", "models_loaded": false, "redis_connected": true}
```

> 注意：首次调用 /api/ai/extract 会自动下载 CLIP 和 BGE 模型（约 1GB），需联网。

## 三、Docker Compose 一键部署

### 3.1 前置要求

- Docker 20.10+
- Docker Compose 2.0+

### 3.2 启动所有服务

```bash
# 构建并启动
docker-compose up -d --build

# 查看状态
docker-compose ps

# 查看日志
docker-compose logs -f app
```

### 3.3 服务端口

| 服务 | 端口 | 说明 |
|------|------|------|
| Web 应用 | 8080 | Spring Boot 主应用 |
| MySQL | 3306 | 数据库 |
| Redis | 6379 | 缓存 + 向量检索 |
| RabbitMQ | 5672 / 15672 | 消息队列 / 管理台 |
| Elasticsearch | 9200 | 搜索引擎 |
| AI 服务 | 8001 | Python AI 微服务 |

### 3.4 停止服务

```bash
docker-compose down

# 清除数据卷（谨慎操作）
docker-compose down -v
```

## 四、降级说明

系统设计为**所有外部服务均可降级**，核心功能不受影响：

| 服务不可用时 | 影响 | 降级行为 |
|-------------|------|----------|
| Python AI | 匹配降级 | 自动使用 TF-IDF 匹配 |
| Redis | 缓存失效 | 直接查询数据库 |
| RabbitMQ | 异步失效 | 改为同步调用 |
| Elasticsearch | 搜索降级 | 使用 MySQL LIKE |
| 邮件服务 | 无邮件 | 跳过邮件发送 |

**最低运行要求**：只需 JDK + MySQL，系统即可完整运行（TF-IDF 匹配 + 基础 CRUD）。
