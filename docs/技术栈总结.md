# 校园失物招领智能匹配系统 V2 - 技术文档

> 大厂简历级 AI+工程闭环 | Spring Boot + 多模态AI匹配 + 向量检索 + 消息队列 + 搜索引擎

---

## 一、项目概述

本系统是面向校园场景的失物招领智能匹配平台，核心亮点是多模态 AI 匹配（CLIP 图像向量 + BGE 文本向量 + GeoHash 位置融合评分），配合 RabbitMQ 异步解耦、Elasticsearch 全文搜索、Item-CF 协同过滤推荐，形成完整的"AI + 工程"闭环。系统采用 Spring Boot + JSP 架构，支持 Docker Compose 一键部署，具备完善的服务降级容灾能力。

### 简历故事线

> 基于 Spring Boot 的校园失物招领智能匹配系统。用户发布失物时，CLIP 模型对图片提取 512 维向量、BGE 模型对文本提取 512 维向量，存入 Redis Stack HNSW 向量索引；匹配时多模态向量融合检索 Top-K（`final_score = 0.4*text + 0.4*image + 0.2*location`），结果经 Redis 缓存加速；匹配命中后通过 RabbitMQ 异步触发 WebSocket 实时通知和邮件提醒；全文搜索接入 Elasticsearch ik 分词；基于 Item-CF 协同过滤实现个性化推荐；使用 JWT+RBAC 鉴权、AOP 记录审计日志；整体通过 Docker Compose 一键部署。

## 五、核心算法

### 5.1 TF-IDF 文本匹配（降级方案）

- **分词**：正向最大匹配（FMM），词典含校园场景词汇
- **IDF 公式**（平滑处理，避免负值）：`idf(t) = log((N+1)/(df+1)) + 1.0`
- **TF-IDF**：`tf-idf(t,d) = tf(t,d) * idf(t)`
- **相似度**：余弦相似度 `cos(A,B) = (A·B) / (|A|*|B|)`

### 5.2 多模态 AI 融合匹配（主方案）

```
final_score = 0.4 * text_sim + 0.4 * image_sim + 0.2 * location_sim
```

- **text_sim**：BGE-Small 提取 512 维文本向量 → Redis Stack KNN 检索 → 余弦相似度
- **image_sim**：CLIP 提取 512 维图像向量 → Redis Stack HNSW 索引 → 余弦相似度
- **location_sim**：GeoHash 编码 → Haversine 球面距离 → 指数衰减 `exp(-d/1000)`
- **权重**：通过 `application.yml` 的 `matching.weights` 配置，`@ConfigurationProperties` 绑定
- **降级**：Python AI 不可用时自动回退 TF-IDF，Redis 缓存降级标记 60 秒

### 5.3 Item-CF 协同过滤推荐

- **物品相似度**：Jaccard 系数（共同浏览用户数 / 总浏览用户数）
- **推荐评分**：`score = Σ(similarity * time_decay)`，`time_decay = exp(-days/30)`
- **冷启动**：无浏览历史时降级为热度推荐（浏览次数最多）
- **缓存**：推荐结果 Redis 缓存 30 分钟，相似度矩阵内存缓存

### 5.4 服务降级策略

| 服务 | 降级条件 | 降级方案 |
|------|----------|----------|
| Python AI | healthCheck 失败 | 回退 TF-IDF 匹配 |
| Redis | isAvailable()=false | 直接查数据库 |
| RabbitMQ | 连接失败 | 同步调用替代异步 |
| Elasticsearch | 模板未注入 | MySQL LIKE 模糊查询 |
| 邮件 | mail.notify.enabled=false | 跳过邮件发送 |
